# Vue 3 Development

## Правила
1. Всегда используй `<script setup lang="ts">` и Composition API.
2. **Роутинг:** В отличие от Nuxt, здесь нет файлового роутинга. Если ты создаешь новую страницу в `src/pages`, ты ОБЯЗАТЕЛЬНО должен импортировать её и добавить маршрут в файл `src/router/index.ts` (или где лежит конфигурация `vue-router`).
3. **Состояние:** Используй Pinia.
4. **Среда:** Проект собирается через Vite. Не используй Nuxt-специфичные функции (например, `useFetch` или `useAsyncData`).

## Работа с документацией (MCP)
- Если ты сомневаешься в синтаксисе Vue 3, Pinia или Vue Router, **не выдумывай код!** Сначала вызови инструменты из MCP-сервера `vue-ecosystem` (например, `vue_api_lookup` или `pinia_docs_search`), прочитай официальный ответ, и только потом пиши реализацию.
- Для поиска готовых композитных функций (composables) всегда сначала ищи их в документации VueUse через этот же MCP-сервер.
