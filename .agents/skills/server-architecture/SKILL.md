# Архитектура Backend (Server)

## Работа с Базой Данных (Drizzle + PostgreSQL)
1. **СУБД и Драйвер:** Проект использует PostgreSQL (через драйвер `pg`) и Drizzle ORM. Любые упоминания SQLite в контексте этого бэкенда неверны.
2. **Схема:** Структура БД и схемы таблиц находятся вне папки `src` (импортируются через `~/../db/schema`). Перед составлением новых запросов или репозиториев обязательно ознакомься со схемой.
3. **ORM:** Запрещено писать сырые SQL-строки (за исключением использования безопасного конструктора `sql` из Drizzle, например, в операциях агрегации или сложных фильтрах). Всегда используй методы Drizzle (`db.select()`, `db.insert()`, `db.query.tableName.findFirst()`).
4. **Мониторинг:** Все тяжелые операции и запросы к БД должны оборачиваться в хелпер `measureDbQuery` из `lib/db-monitoring.ts` для корректного сбора метрик Prometheus.

## Инструменты и Среда (Bun)
- Сервер работает под управлением Bun. Рекомендуется использовать нативные, оптимизированные API Bun вместо стандартных модулей Node.js, где это применимо (например, `Bun.password` для хеширования и проверки паролей в `auth.utils.ts`).
- Точкой входа сервера является `server.ts`, экспортирующий конфигурацию для встроенного веб-сервера Bun (свойства `port`, `hostname`, `fetch`).
- Для валидации кода используй только подходящие бэкенд-инструменты. Использование инструментов, предназначенных исключительно для фронтенда (например, связанных с Vue/Vite), в среде бэкенда запрещено.
- Если возникают сомнения по поводу синтаксиса или возможностей API Bun:
  1. Выполни поиск по документации Bun.
  2. Прочитай локальные справочные файлы проекта, если они доступны в репозитории.
