# Архитектура Монорепозитория (Bun Workspaces)

Этот проект представляет собой монорепозиторий под управлением Bun Workspaces. Код разделен на независимые пакеты в папке `apps/`. При редактировании файлов необходимо строго разграничивать среды выполнения. Смешивание сред запрещено.

## Пакеты (Workspaces)

### 1. `apps/client` (Frontend)
- **Стек:** Vue 3 (Composition API), Vite.
- **Среда выполнения:** Браузер.
- **Правило:** Здесь запрещено импортировать бэкенд-модули, обращаться к базе данных напрямую, использовать API Bun (например, `Bun.serve`, `Bun.file`) или системные пакеты Node.js (`fs`, `path`). Все запросы к серверу осуществляются только через tRPC-клиент или HTTP-запросы.

### 2. `apps/server` (Backend)
- **Стек:** Hono, tRPC, Bun Runtime.
- **Среда выполнения:** Сервер (Bun).
- **Правило:** В этой среде отсутствуют браузерные API (`window`, `document`, `localStorage`). Вся работа с сетью, файлами и внешними API должна строиться на базе Hono, tRPC и возможностей Bun. Здесь не должно быть никакого клиентского интерфейсного кода Vue.

# Обязательные действия

1. Перед завершением работы и написанием ответа ВСЕГДА запускай команду `bun run lint:fix`.
2. После этого ВСЕГДА запускай команду `bun run typecheck`.
3. Если есть ошибки типов или линтера — исправь их самостоятельно.
