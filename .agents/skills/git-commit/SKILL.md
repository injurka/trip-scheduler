# Git Commit Workflow

1. Прочитай текущие изменения через команду `git diff --cached` (или просто `git diff`, если файлы не в индексе).
2. Сгенерируй сообщение для коммита в формате Conventional Commits (feat:, fix:, refactor:, chore:).
3. Сообщение должно четко описывать:
   - В каком слое (kit, domain, feature) были изменения.
   - Что конкретно было сделано.
4. Выведи пользователю предложенный текст коммита и спроси разрешения на выполнение `git commit -m "..."`.
