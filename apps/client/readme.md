# Trip Scheduler - Клиентское приложение

Это клиентская часть веб-приложения и десктопного приложения **Trip Scheduler**, предназначенного для планирования путешествий. Приложение позволяет пользователям не только создавать и детально организовывать маршруты, но и вести интерактивную ленту воспоминаний на основе фотографий и заметок.

## 🚀 Основные технологии

- **Фреймворк:** [Vue.js 3](https://vuejs.org/) (с использованием Composition API и `<script setup>`)
- **Сборка:** [Vite](https://vitejs.dev/)
- **Десктопная версия:** [Tauri](https://tauri.app/)
- **Управление состоянием:** [Pinia](https://pinia.vuejs.org/) (с модульной структурой)
- **Маршрутизация:** [Vue Router](https://router.vuejs.org/)
- **Стилизация:** [Sass (SCSS)](https://sass-lang.com/)
- **UI Компоненты:** Комплексный кастомный UI-kit, [Reka UI](https://www.reka-ui.com/), [Milkdown](https://milkdown.dev/) (для текстового редактора)
- **Карты:** [OpenLayers](https://openlayers.org/)
- **PWA и Offline:** Service Worker, [Workbox](https://developer.chrome.com/docs/workbox/)
- **БД и API:** Динамическое подключение (Tauri SQL / tRPC / Mock)
- **Утилиты:** [VueUse](https://vueuse.org/), [Iconify](https://iconify.design/), [vuedraggable](https://github.com/SortableJS/vuedraggable)
- **Качество кода и Тестирование:**
  - **Линтинг:** [ESLint](https://eslint.org/) с конфигурацией от [@antfu](https://github.com/antfu/eslint-config)
  - **Типизация:** TypeScript (Strict Mode)
  - **Компонентное тестирование:** [Storybook](https://storybook.js.org/) для разработки и визуального тестирования UI-кита.
  - **Unit/E2E Тестирование:** [Vitest](https://vitest.dev/) и [Playwright](https://playwright.dev/).

## ✨ Расширенные возможности

### Планирование и организация

- **Создание путешествий:** Удобный мастер создания поездок с указанием названия, дат и описания.
- **Гибкое планирование по дням:** Каждый день путешествия представляет собой отдельную сущность с возможностью добавления заголовка и заметок.
- **Интерактивные активности:**
  - Добавление активностей с указанием времени начала и окончания.
  - **Drag-and-Drop сортировка:** Легко меняйте порядок активностей перетаскиванием.
  - **Блочная структура:** Каждая активность может состоять из нескольких блоков: текстовые описания (Markdown), фотогалереи и геолокации на карте.
- **Управление статусом:** Отмечайте активности как "Выполнено" или "Пропущено", чтобы отслеживать прогресс.

### Лента воспоминаний

- **Интерактивная хроника:** Помимо плана, для каждого дня доступна "Лента воспоминаний", которая автоматически строится на основе фотографий и заметок.
- **Умная загрузка фото:**
  - При загрузке фотографий приложение анализирует **EXIF-метаданные**.
  - Фотографии с датой и временем съемки автоматически попадают в ленту соответствующего дня и времени.
- **Очередь обработки:** Фото без метаданных попадают в специальную очередь, где пользователь может вручную указать дату и время, а также добавить комментарий.
- **Фоновая загрузка:** Индикация процесса загрузки и обработки файлов без блокировки интерфейса.

### Пользовательский опыт (UX)

- **Кроссплатформенность (Web + Desktop):** Полная функциональность в браузере (PWA) и нативных десктопных приложениях для Linux (deb, AppImage) благодаря Tauri.
- **Продвинутый оффлайн-режим:**
  - **Tauri:** Полноценная работа с данными (чтение и запись) без сети через локальную базу данных SQLite.
  - **Веб (PWA):** Кэширование данных с помощью Service Worker для просмотра информации о поездках оффлайн.
- **Динамическая тема:** Встроенный менеджер тем позволяет переключаться между светлым, темным режимами или создавать и настраивать **собственную цветовую палитру**.
- **Просмотр изображений:** Мощный вьювер изображений с поддержкой просмотра EXIF-метаданных (информация о камере, настройках, GPS) и отображением геолокации на встроенной карте.
- **Адаптивный интерфейс:** Корректное отображение на всех устройствах, от мобильных телефонов до больших десктопных экранов, благодаря кастомному composable `useDisplay`.
- **PWA-уведомления:** Приложение информирует пользователя о доступности новой версии и предлагает обновиться.

### Архитектура и разработка

- **Централизованное управление API:** Кастомный плагин `request` для унифицированной работы с асинхронными запросами, поддерживающий отмену, кэширование и глобальную обработку ошибок.
- **Динамический выбор API:** В зависимости от окружения (`Tauri`, `Web-dev`, `Web-prod`) приложение автоматически переключается между прямыми SQL-запросами, tRPC-клиентом и моковыми данными.
- **End-to-End типизация:** Использование tRPC обеспечивает полную безопасность типов между клиентом и сервером.
- **Кастомный UI-кит:** Богатая библиотека переиспользуемых компонентов (`src/components/01.kit`), разработанная и протестированная в Storybook.

## 📂 Структура проекта

Проект имеет модульную структуру для лучшей организации и масштабируемости, основанную на принципах VSA.

- **`src`**: Основная кодовая база клиентского приложения на Vue.js.
  - **`components`**: Компоненты, организованные по слоям:
    - `01.kit`: Базовые, переиспользуемые UI-компоненты (кнопки, скелетоны, календари, вьювер изображений).
    - `02.shared`: Компоненты и логика, используемые в разных модулях (обертки асинхронных состояний, фоновые эффекты).
    - `05.modules`: Крупные, самодостаточные блоки функциональности (список путешествий, информация о путешествии, лента воспоминаний).
    - `05.layouts`: Компоненты-слои для страниц (`default` с хедером/футером, `empty`).
  - **`pages`**: Компоненты страниц, используемые в Vue Router (например, `/trips`, `/trip/:id`).
  - **`plugins`**: Кастомные плагины Vue для инкапсуляции сквозной логики.
    - `request`: Управляет всеми асинхронными запросами, кэшированием и состоянием загрузки.
    - `theme`: Отвечает за динамическую смену тем и применение CSS-переменных.
    - `session-restore`: Восстанавливает сессию пользователя при запуске приложения.
  - **`shared`**: Общая логика, сервисы, composables и типы.
    - `services/database`: Абстракция для работы с данными, которая динамически выбирает клиент (`real.client.ts` для Tauri, `trpc.client.ts` или `mock.client.ts` для веба) и содержит репозитории для каждой сущности.
  - **`store`**: Глобальные хранилища Pinia (например, `auth.store.ts`, `theme.store.ts`).
- **`src-tauri`**: Код на Rust для сборки десктопного приложения. Отвечает за работу с локальной базой данных SQLite, миграции и сидинг данных.
- **`service-worker`**: Исходный код Service Worker для реализации PWA, управления кэшем и оффлайн-доступа.

## 🚀 Начало работы

### Необходимые условия

- **Bun:** Установленный менеджер пакетов [Bun](https://bun.sh/).
- **Rust:** Для сборки и запуска Tauri-версии необходим установленный [Rust и его toolchain](https://www.rust-lang.org/tools/install).
- **Зависимости Tauri:** Следуйте [гайдлайнам Tauri](https://tauri.app/v1/guides/getting-started/prerequisites) для установки системных зависимостей.

### Установка

```bash
bun install
```

### Запуск

- **Веб-версия для разработки (с моковыми данными или tRPC):**
  ```bash
  bun run dev:web
  ```
- **Десктопная версия для разработки (Tauri):**
  ```bash
  bun run tauri:dev
  ```
- **Просмотр UI-кита в Storybook:**
  ```bash
  bun run dev:storybook
  ```

## 🐳 Сборка Docker

Проект содержит `Dockerfile` для создания готового к развертыванию образа веб-приложения с использованием Nginx.

### Сборка образа

````bash
docker build -t trip-scheduler-client . \
  --build-arg VITE_APP_SERVER_URL=https://api.yourdomain.com \
  --build-arg VITE_APP_API_MODE=trpc
  --build-arg VITE_MAPTILER_KEY=key
```

### Запуск контейнера

```bash
docker run -d -p 1420:1420 --name trip-scheduler-web --restart always trip-scheduler-client:latest
````

Приложение будет доступно по адресу `http://localhost:1420`.

## 🔌 Подключение к данным (API)

Приложение использует гибкую систему подключения к данным, которая адаптируется в зависимости от окружения:

- **Tauri (десктоп):** Прямая работа с локальной базой данных **SQLite** через `tauri-plugin-sql` для максимальной производительности и оффлайн-возможностей.
- **Веб-версия (production):** Взаимодействие с бэкендом через **tRPC**, что обеспечивает полную типизацию API от клиента до сервера.
- **Веб-версия (development):** Может работать в режиме **tRPC** или переключиться на **моковые данные**, если бэкенд недоступен.
